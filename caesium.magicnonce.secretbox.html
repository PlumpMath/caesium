caesium.magicnonce.secretbox

“Magic” nonce schemes for secretbox.

There are two kinds of schemes in this namespace:

  • schemes that take a nonce, but do something with it to make the resulting system safer or easier to use,
  • schemes that produce the nonce for you automatically.

See individual function docstrings for details.

decrypt

(decrypt key ctext)

Decrypts any secretbox message with a prefix nonce.

decrypt-to-buf!

(decrypt-to-buf! out key ctext)

Decrypts any secretbox message with a prefix nonce into the given buffer.

open

(open)

Like decrypt, but with different argument order; analogous to caesium.crypto.secretbox/secretbox-open-easy.

open-to-buf!

(open-to-buf! out ctext key)

Open (decrypt and verify) a nonce-prefixed secretbox message.

secretbox-det

(secretbox-det msg key)

secretbox, with deterministic nonce.

This means the encryption operation requires no new randomness; it is a fully deterministic cryptosystem. This has the necessary consequence that identical plaintexts will map to identical ciphertexts. That has to be acceptable for your protocol!

This scheme does not change the requirements for the key: the key must still be a cryptographically random byte array of appropriate size (caesium.crypto.secretbox/keybytes).

Because the nonce is determined from the plaintext, adding any non-determinism to your message will make the nonce not repeat and hence hide repeated messages from the attacker. A high-resolution timestamp will do that effectively in most cases.

To decrypt, use decrypt or open, depending on which argument order you prefer.

secretbox-nmr

(secretbox-nmr msg nonce key)(secretbox-nmr msg key)

Encrypt a message like secretbox, but nonce-misuse resistant.

This still optionally takes a nonce, but that nonce will be combined with a synthetic nonce. This means that if the nonce incidentally repeats, an attacker will only be able to tell that a message repeated, instead of the usual plaintext disclosure that happens.

If no nonce argument is specified, a random nonce is automatically selected for you, and the nmr scheme is still applied.

secretbox-pfx

(secretbox-pfx msg nonce key)

secretbox, with the given nonce embedded in the ciphertext as a prefix.

To decrypt, use decrypt or open, depending on which argument order you prefer.

secretbox-rnd

(secretbox-rnd msg key)

secretbox, with randomized prefix nonce.

To decrypt, use decrypt or open, depending on which argument order you prefer.
